Alles, was Sie jemals mit KI-Programmierung tun werden, fällt in sechs Kategorien. Nur sechs. Sobald Sie sie erkennen, werden Sie nie wieder verwirrt sein, wonach Sie fragen sollen.

![Sechs Operationen](/code-with-ai/assets/images/diagrams/03-1-six-operations.png)
{: .chapter-diagram}
*Abbildung 3.1: Die sechs Operationen — Alles, was Sie mit KI tun können*
{: .diagram-caption}

Die erste Operation ist das Erstellen von etwas Neuem. Hier beginnen die meisten Projekte. Sie beschreiben, was Sie brauchen — eine Funktion, die die Balkendurchbiegung berechnet, ein Skript, das Dateiformate konvertiert, ein Werkzeug, das Daten verarbeitet — und die KI generiert Code, der vorher nicht existierte. Erstellen ist aufregend, aber auch dort, wo Anfänger oft Schwierigkeiten haben, weil Sie von nichts ausgehen und die Möglichkeiten endlos erscheinen.

Die zweite Operation ist das Lesen und Verstehen von bestehendem Code. Vielleicht haben Sie Code von einem Kollegen geerbt, oder Sie besuchen etwas wieder, das Sie vor sechs Monaten geschrieben haben und nicht mehr wissen, wie es funktioniert. KI ist hervorragend im Erklären. Fügen Sie eine Funktion ein und fragen Sie, was sie macht. Zeigen Sie auf eine verwirrende Zeile und fragen Sie, warum sie da ist. Code lesen ist eine Fähigkeit, bei der selbst erfahrene Programmierer Hilfe brauchen, und KI hat unendliche Geduld beim Erklären.

Die dritte Operation ist das Bearbeiten — bestehenden, funktionierenden Code zu nehmen und ihn zu modifizieren. Sie müssen vielleicht Einheiten von imperial auf metrisch ändern, Ausgabeformate anpassen oder ändern, wie der Code bestimmte Eingaben behandelt. Bearbeiten ist oft sicherer als von Grund auf neu zu erstellen, weil Sie auf einem Fundament aufbauen, das bereits funktioniert. Sie wissen, dass die Grundstruktur solide ist; Sie passen nur Details an.

Die vierte Operation ist das Ausführen des Codes, und dies ist die einzige, die KI nicht für Sie tun kann. Sie müssen den Code selbst ausführen und beobachten, was passiert. Das ist Ihr Realitätscheck. Code, der perfekt aussieht, könnte in der Praxis versagen. Funktionen, die vollständig erscheinen, könnten falsche Antworten liefern. Code ausführen verbindet die abstrakte Welt der Anweisungen mit der konkreten Welt der Ergebnisse. Überspringen Sie diesen Schritt niemals.

Die fünfte Operation ist das Beheben von Fehlern. Wenn Code versagt — und das wird er — gehen Sie in den Debugging-Modus. Der beste Ansatz ist direkt: Fügen Sie die Fehlermeldung genau so ein, wie sie erscheint, fügen Sie den relevanten Code hinzu und bitten Sie die KI, ihn zu reparieren. Fehlermeldungen sehen oft einschüchternd aus, zeigen aber normalerweise direkt auf das Problem. Je mehr Kontext Sie darüber geben, was Sie erwartet haben im Vergleich zu dem, was passiert ist, desto schneller kann die KI das Problem diagnostizieren.

Die sechste Operation ist das Erweitern der Funktionalität. Ihr grundlegender Code funktioniert, aber jetzt möchten Sie, dass er mehr macht. Vielleicht sollte Ihr Rechner auch Ergebnisse in eine Datei speichern, oder Ihr Datenverarbeiter sollte ein neues Dateiformat unterstützen, oder Ihr Analysewerkzeug sollte Diagramme generieren. Erweitern bedeutet, Fähigkeiten inkrementell hinzuzufügen, auf bewährten Grundlagen aufzubauen, anstatt von vorne anzufangen.

Wenn Sie feststecken und nicht wissen, wonach Sie fragen sollen, identifizieren Sie, welche Operation Sie brauchen. Wenn Ihr Prompt keine guten Ergebnisse liefert, prüfen Sie, ob Sie eindeutig eine Operation zur Zeit durchführen. Dieses Vokabular — Erstellen, Lesen, Bearbeiten, Ausführen, Beheben, Erweitern — macht die Kommunikation mit KI präzise und effektiv.

<div class="key-insight">
<strong>Wichtige Erkenntnis:</strong> Diese sechs Operationen zu kennen, gibt Ihnen ein Vokabular für die Arbeit mit KI. Sie „bitten nicht einfach die KI um Hilfe" — Sie führen spezifische Operationen durch, die spezifische Muster haben.
</div>

## Probieren Sie es selbst aus

Üben Sie jede Operation:

- **Erstellen:** „Schreibe eine Funktion, die die Schraubenspannung aus Kraft und Fläche berechnet"
- **Lesen:** „Erkläre, was dieser Code macht: `result = [x*2 for x in range(10)]`"
- **Bearbeiten:** „Ändere diese Funktion, um Millimeter statt Meter zu verwenden"
- **Beheben:** „Ich bekomme ‚TypeError: unsupported operand type' — hier ist mein Code..."
- **Erweitern:** „Füge die Möglichkeit hinzu, Ergebnisse in eine CSV-Datei zu speichern"
- **Lesen:** „Warum läuft diese Schleife endlos?"

---

## Wichtigste Erkenntnis

Jede Interaktion mit KI zum Programmieren ist eine von sechs Operationen: Erstellen, Lesen, Bearbeiten, Ausführen, Beheben oder Erweitern. Wenn Sie verwirrt sind, wonach Sie fragen sollen, identifizieren Sie, welche Operation Sie brauchen. Wenn Ihr Prompt nicht funktioniert, stellen Sie sicher, dass Sie eindeutig eine Operation zur Zeit durchführen. Dieses Vokabular macht die Kommunikation mit KI präzise und effektiv.

In Teil 2 werden wir die Fähigkeiten entwickeln, die Sie brauchen, um diese Operationen gut durchzuführen — beginnend damit, wie man große Probleme in kleine Teile zerlegt.
